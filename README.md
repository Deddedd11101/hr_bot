
## HR Bot — демо первого дня сотрудника

Демо‑проект для онбординга нового сотрудника через Telegram‑бота и простую административную панель.

Флоу реализует автоматическую отправку напоминаний в первый рабочий день:
- в 10:00 — старт рабочего дня и информация о встрече с HR;
- в 11:00 — напоминание заполнить анкету;
- далее — отправка документов на подпись;
- в 12:00 — напоминание о встрече с руководителем;
- в 13:00 — напоминание об обеде;
- в 14:00 — напоминание о доступах;
- в 15:00 — документ со структурой компании;
- в 16:00 — ссылка на бизнес‑процессы;
- в 17:00 — напоминание про анализ первого дня;
- в 18:00 — анкета и чек‑лист по первому дню.

### Стек

- **Backend / API**: FastAPI
- **БД**: SQLite + SQLAlchemy
- **Планировщик задач**: APScheduler
- **Telegram‑бот**: Aiogram
- **Админ‑панель**: простые HTML‑шаблоны на Jinja2 поверх FastAPI

### Быстрый старт

1. **Зависимости**

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

2. **Переменные окружения**

Создайте файл `.env` в корне проекта:

```bash
TELEGRAM_BOT_TOKEN=ваш_токен_бота
# опционально:
# DEMO_MODE=true          # включает быстрый режим
# DEMO_STEP_MINUTES=1     # интервал между шагами флоу в минутах
# FILE_STORAGE_DIR=./storage/employee_files
```

3. **Запуск админ‑панели**

```bash
uvicorn app.main:app --reload
```

Админка будет доступна по адресу `http://localhost:8000`:
- список сотрудников;
- форма создания нового сотрудника и запуска флоу первого дня.

4. **Запуск Telegram‑бота и планировщика**

В отдельном терминале:

```bash
python -m app.bot_runner
```

Этот процесс:
- поднимает Telegram‑бота;
- запускает планировщик APScheduler;
- при создании нового сотрудника по API или через админку планирует цепочку уведомлений на первый день.

### Быстрый демо‑режим

Чтобы не ждать реальные часы с 10:00 до 18:00, можно включить быстрый режим:

1. В `.env` добавьте:

```bash
DEMO_MODE=true
DEMO_STEP_MINUTES=1  # каждый следующий шаг флоу через 1 минуту
```

2. Перезапустите оба процесса:

```bash
# терминал 1
uvicorn app.main:app --reload

# терминал 2
python -m app.bot_runner
```

3. В админке (`http://localhost:8000`) создайте нового сотрудника с любой датой первого дня
   (можно поставить сегодняшнюю). После этого:
- бот каждые `DEMO_STEP_MINUTES` минут будет отправлять следующее сообщение по сценарию первого дня;
- лог отправленных событий сохраняется в БД `hr_bot.db` (таблица `onboarding_events`).

### Что уже готово в демо

- Модель сотрудника с Telegram `user_id` и датой первого рабочего дня.
- Стандартный флоу уведомлений для первого дня (10:00–18:00).
- REST‑API и простая HTML‑админка для управления сотрудниками.
- Telegram‑бот, который отправляет запланированные сообщения сотрудникам.
- Дополнительные поля кандидата/сотрудника: должность, зарплатные ожидания, согласия, тестовое, статус, заметки.
- Приём входящих файлов в Telegram и сохранение в карточке сотрудника.
- Загрузка файлов из админки, скачивание и отправка файлов сотруднику в Telegram.
- Локальное файловое хранилище (`FILE_STORAGE_DIR`, по умолчанию `./storage/employee_files`).
