# HR Bot Admin

Админка и Telegram-бот для HR-процессов: работа с кандидатами и сотрудниками, запуск сценариев, массовые рассылки, опросы, файловые вложения и уведомления по действиям пользователей.

## Стек

- Backend: FastAPI
- База данных: SQLite + SQLAlchemy
- Telegram-бот: Aiogram
- Планировщик: APScheduler
- Интерфейс админки: Jinja2 + HTML/CSS
- Работа с файлами: локальное хранилище `storage/employee_files`

## Быстрый старт

1. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Создайте `.env` в корне проекта:

```bash
TELEGRAM_BOT_TOKEN=your_bot_token
FILE_STORAGE_DIR=./storage/employee_files
```

3. Запустите веб-интерфейс:

```bash
uvicorn app.main:app --reload
```

4. В отдельном терминале запустите бота и планировщик:

```bash
python -m app.bot_runner
```

По умолчанию админка доступна на [http://localhost:8000](http://localhost:8000).

## Что есть в админке

Верхнее меню состоит из 6 вкладок:

- `Кандидаты`
- `Сотрудники`
- `Массовые действия`
- `Сценарии`
- `Опросы`
- `Настройки`

## Вкладка `Кандидаты`

Список показывает только сотрудников со статусом `Кандидат`.

В таблице доступны:

- ID
- ФИО
- Telegram
- должность
- дедлайн тестового задания
- название запланированного сценария
- переход в карточку кандидата

В карточке кандидата можно:

- редактировать ФИО, Telegram и предварительную дату выхода
- указывать желаемую должность
- вручную выставлять текущий этап работы кандидата
- хранить ожидания по зарплате
- задавать дедлайн тестового задания
- хранить согласие на ПДн кандидата
- вести заметки HR
- загружать и скачивать файлы
- отправлять файл кандидату в Telegram
- сохранять персональную ссылку на оффер
- планировать запуск сценария
- запускать сценарии вручную
- смотреть историю ручных запусков с разворачиванием списка

## Вкладка `Сотрудники`

Список показывает всех, кроме кандидатов.

В таблице доступны:

- ID
- ФИО
- Telegram
- должность
- статус
- первый рабочий день
- название запланированного сценария
- переход в карточку сотрудника

В карточке сотрудника можно:

- редактировать ФИО и Telegram
- указывать дату выхода на работу и дату рождения
- хранить должность
- задавать рабочую почту и рабочие часы
- задавать Telegram ID руководителя
- задавать Telegram ID наставника по адаптации
- задавать Telegram ID наставника по ИПР
- вручную выставлять статус сотрудника
- хранить согласие на ПДн сотрудника
- вести заметки HR
- работать с файлами и отправкой файлов в Telegram
- хранить персональную ссылку на оффер
- планировать запуск сценария
- запускать сценарии вручную
- смотреть историю ручных запусков с разворачиванием списка

## Вкладка `Массовые действия`

Раздел разбит на раскрывающиеся секции:

- `Сценарии`
- `Сообщения`
- `Опросы`

Для сценариев доступны:

- планировщик запуска
- ручной запуск
- выбор сценария
- выбор даты и времени
- выбор получателей по должности
- выбор конкретного сотрудника или кандидата
- таблица запланированных запусков
- таблица истории ручных запусков
- удаление запланированных запусков

Для сообщений доступны:

- планировщик отправки
- ручная отправка
- текст сообщения
- теги `{name}` и `{full_name}`
- тег документа `{doc:Оффер}`
- emoji picker
- выбор получателей по должности
- выбор конкретного сотрудника или кандидата
- таблица запланированных сообщений
- таблица истории ручных отправок
- удаление запланированных сообщений

Для опросов доступны:

- планировщик запуска опроса
- ручной запуск опроса
- выбор опроса
- выбор получателей по должности
- выбор конкретного сотрудника или кандидата
- таблица запланированных запусков
- таблица истории ручных запусков
- удаление запланированных запусков

## Вкладка `Сценарии`

Список сценариев поддерживает:

- создание нового сценария
- сортировку сценариев перетаскиванием
- копирование сценария
- редактирование
- удаление

В таблице сценариев видны:

- ID
- название
- привязка к должности
- режим запуска
- описание

## Редактор сценария

Редактор сценария поддерживает:

- название, описание, привязку к должности и к конкретному сотруднику
- выбор режима запуска сценария
- компактные сворачиваемые карточки шагов
- сортировку шагов перетаскиванием
- отправку текста
- отправку документа или изображения
- теги `{name}`, `{full_name}`, `{doc:Оффер}`
- emoji picker
- настройку времени отправки шага
- сохранение ответа в поле карточки сотрудника
- варианты ответа:
  - без ответа
  - текст
  - файл
  - кнопки
  - ветвление
- уведомление при отправке шага
- уведомления по нажатию кнопок

### Ветвления в сценариях

Для шага с типом `Ветвление` доступны:

- список кнопок
- отдельная карточка ветки на каждую кнопку
- сообщение после выбора кнопки
- вложение для ветки
- следующий шаг после ветки:
  - без ответа
  - текст
  - файл
  - кнопки
  - цепочка шагов
  - переход к другому сценарию
- уведомление при отправке сообщения ветки
- уведомление при выборе кнопки ветки

### Цепочки шагов внутри ветвления

Для ветки с типом `Цепочка шагов` доступны:

- вложенный редактор шагов
- добавление и удаление шагов цепочки
- ветвление внутри шагов цепочки
- время отправки для шагов цепочки
- сохранение ответа в поле сотрудника
- возврат в родительский сценарий после завершения цепочки

## Вкладка `Опросы`

Раздел визуально повторяет сценарии, но используется для анкет и опросников.

Список опросов поддерживает:

- создание
- сортировку перетаскиванием
- копирование
- редактирование
- удаление
- выгрузку результатов в Excel

## Редактор опроса

Редактор опроса поддерживает:

- название и описание опроса
- привязку к должности
- привязку к конкретному сотруднику или кандидату
- список вопросов вместо шагов сценария
- текст вопросов
- вложения
- теги `{name}`, `{full_name}`, `{doc:Оффер}`
- emoji picker
- варианты ответа:
  - без ответа
  - текст
  - файл
  - кнопки
  - ветвление

Опросы запускаются из вкладки `Массовые действия`.

## Результаты опросов

При прохождении опросов ответы сохраняются в БД по пользователям и вопросам.

Excel-выгрузка формируется так:

- одна строка = один сотрудник или кандидат
- отдельные столбцы = текст сообщений вопросов
- в ячейках лежат ответы пользователя

## Вкладка `Настройки`

Раздел включает несколько блоков.

### Общие настройки уведомлений

Можно настроить:

- имя HR
- основной Telegram user_id
- дополнительные Telegram user_id для уведомлений
- включение уведомлений:
  - по завершению сценариев
  - по получению тестового задания
  - по действиям пользователей в боте

### Меню чат-бота

Можно настроить интерфейс кнопок Telegram-бота:

- создавать наборы кнопок
- редактировать название набора
- удалять наборы
- добавлять кнопки в набор
- менять подпись кнопки
- выбирать действие кнопки:
  - неактивна
  - запуск сценария
  - переход к другому набору кнопок
- сохранять настройки всего меню

### Доступ в админку

Для администратора доступно:

- создание аккаунтов
- выбор роли
- включение и отключение аккаунта
- обновление существующих пользователей

## Файлы и документы

Система поддерживает:

- загрузку файлов в карточку кандидата или сотрудника
- отправку файлов пользователю в Telegram
- прикрепление файлов к шагам сценария
- прикрепление файлов к веткам и цепочкам
- подстановку персональной ссылки на оффер через тег `{doc:Оффер}`

## Telegram-бот

Бот умеет:

- регистрировать пользователя по `/start`
- связывать пользователя с карточкой сотрудника
- запускать сценарии
- проходить ветвления и цепочки шагов
- принимать текстовые ответы
- принимать файлы
- отправлять документы и изображения
- работать с меню кнопок из настроек
- отправлять служебные уведомления по действиям пользователя
