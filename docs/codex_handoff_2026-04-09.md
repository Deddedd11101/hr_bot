# Codex Handoff — 2026-04-09

Этот файл фиксирует контекст по основным изменениям за последние пару дней, чтобы можно было быстро продолжать работу без чтения длинного треда.

## Текущая ветка

- `codex/scenario-workspace-ui-foundation`

## Что оставили как source of truth

- Классический редактор сценариев остаётся рабочим fallback-вариантом.
- Классический список сотрудников тоже остаётся как fallback.
- Новые экраны развиваются параллельно, без удаления рабочей классики.

## Что сделали по сценариям

### 1. Отказались от старого visual editor на React Flow

- Пробовали отдельный visual editor сценариев.
- Подход оказался слишком сырым по UX и логике.
- Его убрали из проекта, чтобы не мешался.

### 2. Собрали новый `workspace v2`

Новый маршрут:

- `/app/flows/workspace-v2`

Состояние:

- это основной новый React-экран для работы со сценариями;
- старый `/app/flows/workspace` больше не нужен как отдельный экран и должен вести в `v2`;
- классический редактор сценариев оставили как fallback.

### 3. Что умеет `workspace v2`

- список сценариев слева;
- поиск по сценариям;
- создание сценария;
- массовые действия по сценариям;
- reorder сценариев;
- основной поток шагов;
- создание шага в основном потоке;
- создание ветки;
- создание шага цепочки;
- удаление шага / ветки / chain-step;
- редактирование шага в правой панели `Детали`;
- вложения шага;
- emoji picker;
- новые popover-выпадашки в деталях;
- nested branching:
  - ветка может иметь `Ветвление`;
  - внутрь такой ветки теперь можно проваливаться;
- chain внутри ветки:
  - можно открыть;
  - можно добавлять шаги цепочки.

### 4. Что изменили по логике интерфейса сценариев

- тип `Выбор кнопками` убран из нового UI как отдельная полезная сущность;
- в новом интерфейсе опираемся на:
  - обычный шаг;
  - `Ветвление`;
  - `Цепочка шагов`;
- правую панель `Детали` перестроили в одну колонку;
- блок `Уведомление для шага` нужно держать закрытым по умолчанию;
- у каждой колонки свой скролл;
- правый блок деталей должен оставаться привязанным к экрану.

### 5. Что ещё не доделано по сценариям

- уведомления `при ответе пользователя`:
  - пока это только обсуждённая идея, не реализовано;
- теги для уведомлений на основе ответа пользователя:
  - например `{response_text}`, `{response_file_name}` — ещё не делались;
- reorder глубже основного потока:
  - ветки;
  - шаги цепочки;
- более структурный UX для веток:
  - сейчас кнопки ветвления всё ещё редактируются через textarea;
- общий polish новых popover/select-контролов.

## Что сделали по сотрудникам

Новый маршрут:

- `/app/employees`

Состояние:

- страница пересобрана на новом frontend-стеке;
- старая версия сотрудников остаётся как fallback;
- старую промежуточную React-версию без нового UI-слоя убрали.

### Что умеет новый список сотрудников

- переключение `Сотрудники / Кандидаты`;
- карточный и табличный режим;
- поиск;
- фильтрация;
- inline-форма создания;
- выбор типа записи при создании:
  - сотрудник;
  - кандидат;
- выбор статуса/этапа при создании;
- более спокойный UI в одном стиле со сценариями;
- широкая рабочая область без старой зажатой колонки.

### Что важно по UX сотрудников

- явную ссылку на классический список убрали из нового UI;
- список переключается мягче, чем раньше;
- grid карточек и табличный режим уже работают;
- текущая страница является новым основным React-вариантом списка.
